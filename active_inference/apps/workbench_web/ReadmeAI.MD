# apps/workbench_web/ ‚Äî ReadmeAI

## Purpose

The **Phoenix LiveView workbench UI** and **episode orchestrator**. The only app aware of both the agent plane and the world plane ‚Äî runs them side-by-side across the Markov blanket, publishes events to `WorldModels.Bus`, and renders live visuals (composition canvas, Glass Engine, labs, guide, equation registry, model taxonomy).

## Telemetry / signals / events emitted here

- `WorkbenchWeb.Episode` publishes these `WorldModels.Event.type` values to `WorldModels.Bus`:
  - `"world.observation"` ‚Äî Engine obs packet (initial + after each apply)
  - `"agent.perceived"` ‚Äî post-Perceive state beliefs
  - `"agent.planned"` ‚Äî F, G, policy posterior
  - `"agent.action_emitted"` ‚Äî chosen action
  - `"world.terminal"` ‚Äî terminal condition reached

- Phoenix endpoint emits `[:phoenix, :endpoint, ...]` events (standard).
